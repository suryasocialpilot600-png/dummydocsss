This guide helps you authenticate and make your first API request to manage users.

## Prerequisites

Before you begin, make sure you have:

- A SocialPilot account
- An API Key from your **Settings** dashboard

## Get started

<Steps>
  <Step title="Generate authentication token">
    Retrieve an authentication JSON Web Token (JWT) using your API key. Make a `GET` request to `/generate-token`. Include the `userId` in your query parameters.

    ```bash
    curl -X GET "https://staging-api.socialpilot.co/generate-token?userId=123" \
      -H "apiKey: YOUR_API_KEY"
    ```

    Save the token from the response. It expires after the duration specified in `expiresIn`.
  </Step>

  <Step title="Authenticate requests">
    Use the retrieved JWT token in the authentication header for all user management endpoints. Set the header key based on your authentication mechanism.
  </Step>

  <Step title="Create a team member">
    Send a `POST` request to `/user/create` to provision a new user. You can assign roles such as `OWNER`, `ADMIN`, `MANAGER`, `CONTENT_SCHEDULER`, or `CLIENT`.

    ```bash
    curl -X POST "https://staging-api.socialpilot.co/user/create" \
      -H "apiKey: YOUR_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "email": "manager@myorg.com",
        "accounts": [1234, 6789],
        "firstName": "Julie",
        "lastName": "Lewis",
        "role": "MANAGER",
        "access": {
          "analyticsAccess": true,
          "inboxAccess": true,
          "libraryAccess": true
        }
      }'
    ```
  </Step>
</Steps>
